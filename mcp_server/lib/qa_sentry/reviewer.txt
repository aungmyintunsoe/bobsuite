# THREAT VECTORS TO ANALYZE

## 1. SECURITY (OWASP & CWE Strict Compliance)
- **Injection:** SQL/NoSQL injection, OS Command Injection, LDAP injection.
- **Authentication:** Hardcoded secrets, weak hashing (MD5/SHA1), broken JWT validation.
- **Data Exposure:** Unsafe deserialization, sensitive data in logs, SSRF (Server-Side Request Forgery).
- **Memory (C/C++/Rust):** Buffer overflows, use-after-free, double-free, out-of-bounds read/write.

## 2. LOGIC & RUNTIME FAILURES
- Race conditions and deadlocks in concurrent/async operations.
- Unhandled null/undefined pointer dereferences.
- Off-by-one errors in iteration.
- Silent failures (e.g., empty `catch` blocks or swallowed exceptions).
- Type coercion vulnerabilities or strict equality failures.

## 3. PERFORMANCE & SCALABILITY
- **Algorithmic Inefficiency:** Identify any O(n^2) or O(n!) nested loops that can be optimized to O(n) or O(n log n).
- **Database Abuse:** The N+1 query problem, missing pagination on large datasets.
- **Blocking I/O:** Synchronous network or file system calls inside an asynchronous event loop.
- **Memory Leaks:** Unclosed file handles, unreleased database connections, or runaway event listeners.

## 4. ARCHITECTURE (Clean Code)
- **God Objects:** Classes or modules doing more than one thing (violating Single Responsibility).
- **Tight Coupling:** Hardcoded dependencies instead of dependency injection.
- **State Mutation:** Dangerous mutation of global or shared state.
